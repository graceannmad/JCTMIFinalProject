# Data Model — JCTMI Final Project

**No database.** All entities are either static files committed to the repo or TypeScript types that exist only within a request/response cycle.

---

## Static Data Files

### `/data/poets.json`

Hand-authored. One entry per featured poet. Updated manually when adding a new poet.

```ts
type PoetProfile = {
  id: string              // "judah-halevi" — used as foreign key in corpus
  name: string            // "Judah Halevi"
  dates: string           // "c. 1075–1141"
  context: string         // 2–3 sentence historical background shown on results page
  voiceDescription: string // persona notes injected into Claude system prompt
                           // e.g. "Speaks with longing and spiritual intensity.
                           //       Uses nature imagery (doe, gazelle, sea).
                           //       Blends sacred and erotic registers."
  portraitUrl?: string    // optional illustration URL
}
```

**Example entry:**
```json
{
  "id": "judah-halevi",
  "name": "Judah Halevi",
  "dates": "c. 1075–1141",
  "context": "Born in Al-Andalus, Halevi wrote in Hebrew but was steeped in Arabic poetic forms. His secular love poetry fuses Neoplatonic longing with vivid sensory imagery — the beloved as a doe, the heart as a battlefield.",
  "voiceDescription": "Warm, aching, and intensely present. Speaks directly to the beloved. Uses animal imagery (doe, gazelle), natural light, and the body as metaphor. Oscillates between ecstasy and grief. Never ironic.",
  "portraitUrl": null
}
```

### `/data/corpus.json`

Generated by the OCR pipeline (`/scripts/ocr.ts`). One entry per poem or poem section. Do not hand-edit — regenerate from source scans.

```ts
type CorpusChunk = {
  poet: string       // must match a PoetProfile.id
  poemTitle: string  // "The Sensitive Doe"
  text: string       // English translation text only
  sourcePage: number // original scan page number, for attribution
}
```

**Example entry:**
```json
{
  "poet": "judah-halevi",
  "poemTitle": "The Sensitive Doe",
  "text": "O graceful doe, pity this heart in which you have dwelled all your life...",
  "sourcePage": 342
}
```

## Request-Scoped Types

These types exist only within a single API request. Nothing is persisted.

### Quiz input (browser → `/api/lyrics` and `/api/generate`)

```ts
type QuizInput = {
  artist: string        // free text, e.g. "Frank Ocean"
  song: string          // free text, e.g. "Ivy"
  lyricChoice: string   // text of the selected lyric option (Q3)
  excerptChoice: string // text of the selected poetry excerpt (Q4)
  moodChoice: string    // text of the selected mood option (Q5)
}
```

### Lyrics data (internal, `/api/lyrics` response)

```ts
type LyricsData = {
  artistLyrics: string[]  // lyrics of artist's top songs (up to 5)
  songLyrics?: string     // lyrics of the specific entered song
  geniusMiss?: boolean    // true if Genius couldn't find the artist/song
}
```

### Playlist item (internal + results page)

```ts
type PlaylistItem = {
  title: string           // song title — verified against Spotify
  artist: string          // artist name — verified against Spotify
  spotifyUrl: string      // direct Spotify track URL
  justification: string   // 2–3 sentences in poet's voice
  poemReference?: string  // optional quote from the poet's own work
}
```

### Results payload (`/api/generate` response → browser)

```ts
type ResultsPayload = {
  poet: PoetProfile
  matchExplanation: string  // first-person poet voice, ~150 words
  historicalContext: string // drawn from poet.context, possibly expanded by Claude
  playlist: PlaylistItem[]  // 6–10 items (may be fewer if Spotify verification fails)
}
```

## Claude Tool Schemas

These are the JSON schemas passed to Claude's tool use API to guarantee structured output.

### `match_poet` tool (Call 1 output)

```json
{
  "name": "match_poet",
  "description": "Return the matched poet and the reasoning for the match.",
  "input_schema": {
    "type": "object",
    "properties": {
      "matchedPoetId": {
        "type": "string",
        "description": "The id field of the matched poet from the provided poet list."
      },
      "matchReasoning": {
        "type": "string",
        "description": "2–3 sentences explaining why this poet's style matches the user's musical taste. Written in third person for internal use."
      }
    },
    "required": ["matchedPoetId", "matchReasoning"]
  }
}
```

### `generate_playlist` tool (Call 2 output)

```json
{
  "name": "generate_playlist",
  "description": "Return the match explanation, historical context, and playlist — all written in the poet's first-person voice.",
  "input_schema": {
    "type": "object",
    "properties": {
      "matchExplanation": {
        "type": "string",
        "description": "~150 words in first-person poet voice explaining to the user why they were matched."
      },
      "historicalContext": {
        "type": "string",
        "description": "2–3 sentences of historical context about the poet."
      },
      "playlist": {
        "type": "array",
        "items": {
          "type": "object",
          "properties": {
            "title":          { "type": "string" },
            "artist":         { "type": "string" },
            "justification":  { "type": "string", "description": "2–3 sentences in poet's first-person voice." },
            "poemReference":  { "type": "string", "description": "Optional: a short quote from one of the poet's own poems, relevant to this song." }
          },
          "required": ["title", "artist", "justification"]
        },
        "minItems": 8,
        "maxItems": 10
      }
    },
    "required": ["matchExplanation", "historicalContext", "playlist"]
  }
}
```
